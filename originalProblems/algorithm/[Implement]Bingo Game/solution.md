# Solution

题意相当明白了，一道模拟题。

## Start with BruteForce
首先是朴素的算法，对于每个念出的数字，直接暴力遍历查找打上标记，每次念完后暴力检查是否全打完了标记。
复杂度$(Tknm)$，显然会$TLE$到飞起。

## Continue with STL
随后考虑优化，暴力的查找打标记显然可以优化为$\log n \times m$的操作。  
具体的话，就是用map来记录每个数字的位置，因为互不重复，所以是可行的。  
那么每次命中后检查是否在游戏范围内，可以记录一个$tot$代表命中数量，当$tot$达到游戏范围内位置总数的时候就可以**Bingo**了。  
需要注意的是，念出的数字有可能重复，因此每次命中都要将该位置从游戏范围内排除。  
那么复杂度就是$O(Tk\log{n \times m})$，可以通过本题。  

# Conclusion
作为T3送分题，这道题还是相当友善的。简单地优化一下就可以通过，即使是最简单的模拟方法也可以得到$50$分的好分数。
虽然可能因为常数不同而有所不同，但大体上得分还是可观的。
